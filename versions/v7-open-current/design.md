# Fugue Landing Page v7

## Working name

`Open Current`

## Intent

v7 starts from one precise complaint about v6:

- the water signal was strong
- but it still lived inside a right-side object

That made the effect feel like a contained demo asset instead of the atmosphere of the whole launch moment.

So v7 keeps the same cathode / watery CRT mechanism, but changes the composition rule completely:

**the signal should flood the opening viewport, and the copy should sit inside that field rather than beside a framed screen.**

## Why v6 was not enough

v6 improved taste and restraint, but the hero still had one remaining enclosure:

- the spectacle was boxed into a machine-like slab
- the right column still behaved like a featured object
- the opening feeling was “look at this screen” instead of “step into this current”

For Fugue, the second reading is stronger.

The product promise is a route across environments.
The hero should therefore feel environmental, not product-shot-only.

## New direction

### Core idea

The first viewport becomes:

- a full-bleed current field
- one huge statement in the lower left
- a floating route annotation rail on the right
- a runway of route logic sitting on the same surface

The memorable thing is no longer the framed slab.

It becomes:

**the sense that the entire page is carrying the signal.**

### Design lane

- full-width atmospheric hero
- minimal copy over a flooded visual field
- no boxed hero hardware
- floating note modules instead of one enclosed screen
- line-based editorial structure below the fold

## Visual system

### Typography

- Display: `Syne`
- Body: `Manrope`
- Technical labels: `IBM Plex Mono`

The typography remains forceful and compressed, but v7 gives the layout more negative space so the display type can breathe against the background field.

### Palette

- Base: near-black blue charcoal
- Text: warm paper white
- Accent: cold steel blue

The palette stays disciplined.
The goal is not more color.
The goal is stronger atmosphere.

### Material strategy

- the hero uses open layers instead of a framed shell
- the proof terminal remains the primary enclosed object below the fold
- route notes float as translucent markers over the current

This follows a stricter rule than v6:

**do not put the most interesting effect inside a box unless the box is the point.**

## Motion strategy

### Signature motion

The Unicorn scene remains the signature motion layer, but v7 changes where that motion lives:

- the watery CRT field spans the whole opening section
- pointer movement shifts the field subtly, not a device mockup
- stagger reveals stay on the copy and route rail only

That makes the hero feel broader and less object-bound.

### Reduced motion

If `prefers-reduced-motion` is enabled:

- the Unicorn scene is skipped
- the hero keeps a static atmospheric fallback
- tilt and staged entrances collapse cleanly

## Information architecture

The page sequence remains lean:

1. Hero statement
2. Route thesis
3. Current boundary
4. Quickstart proof
5. Auth handoff

The conversion logic is unchanged:

- understand the route immediately
- inspect the real API surface
- accept auth as the next actual layer

## Skill-driven decisions

This version follows the same local frontend skill family, but applies it in a more radical way:

- `design-taste-frontend`: push asymmetry without falling back to generic SaaS cards
- `high-end-visual-design`: treat the hero as an atmospheric film frame, not a boxed component
- `bolder`: move the effect from one object to the whole viewport
- `distill`: keep the copy short enough that the field still reads
- `animate`: preserve one hero motion moment and avoid scattering gimmicks everywhere
- `page-cro`: keep the primary CTA and proof section near the fold

## Success criteria

If v7 works, the user should remember:

- the water signal filling the opening screen
- the headline sitting inside the current, not next to a device
- the right-side route notes feeling like live annotations on the field
