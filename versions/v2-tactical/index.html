Primary route

Three moves explain the whole product.

Fugue is easier to trust when the path is explicit: import code, run on shared infrastructure, then move onto your own machine without changing the control surface.

01

Import a public GitHub repository

repo intake

Bring code in, choose a branch and build mode, and let Fugue queue the first deploy without making you bootstrap your own cluster first.

  • repo_url / github.com/example/static-site
  • branch / main
  • builder / auto
02

Deploy onto the shared runtime

managed path

Use the managed shared runtime as the shortest path to a public URL, while build steps, deploy state, logs, and audit remain visible from the start.

  • runtime_id / runtime_managed_shared
  • operation / build, deploy, route
  • visibility / logs + audit trail
03

Attach a VPS and migrate later

exit path

Generate a node key, attach your own machine, confirm heartbeat, and migrate the workload without changing the app model or relearning the product surface.

  • node key / issue once
  • heartbeat / runtime stable
  • migrate / shared -> attached

Object model

Five objects explain the control plane.

Once the model is visible, the product reads like a system instead of a black box: workspace, project, app, runtime, and operation are the pieces that matter.

workspace Tenant boundary
project Default project
app Imported app
runtime Shared or attached
operation Deploy, scale, migrate
Workspace

Maps to the tenant shell and scopes the product boundary.

Project

Groups imported apps without flattening everything into one bucket.

App

The imported workload keeps the same identity across shared and attached runtimes.

Runtime

Can be managed shared infrastructure or a machine you attach later.

Operation

First-class execution record for deploy, migrate, and other runtime changes.

Quickstart 
export FUGUE_BASE_URL="https://api.fugue.pro"

curl -sS "${FUGUE_BASE_URL}/healthz"

curl -sS "${FUGUE_BASE_URL}/v1/apps/import-github" \
  -H "Authorization: Bearer <tenant-api-key>" \
  -H "Content-Type: application/json" \
  -d '{
    "repo_url":"https://github.com/example/static-site",
    "runtime_id":"runtime_managed_shared"
  }'
