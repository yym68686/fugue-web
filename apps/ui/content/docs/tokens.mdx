---
title: Semantic tokens
description: The primitive to semantic to component token hierarchy.
---

Components consume `background`, `foreground`, `card`, `popover`, `primary`, `secondary`, `muted`, `accent`, `destructive`, `border`, `input`, `ring`, sidebar and status tokens. Light and dark themes change semantic variables, never component markup.

Fugue brand expression belongs in semantic token values and product composition, not a second visual runtime.
